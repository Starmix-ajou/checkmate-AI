### 사용자 입력으로부터 기능 정의서 출력
# 사용자 입력을 받는 엔드포인트 ('/generate') 정의
# 기능 '정의서' 생성 및 수정은 ('/generate/definition') 엔드포인트에서 수행하고,
# 기능 '명세서' 자동 생성은 ('/generate/specification') 엔드포인트에서 수행
# GPT-4o API를 사용해서 기능 정의서와 명세서 생성 및 수정을 지원

### 진행 과정
# 1. 사용자 입력 (자연어)
# 테스트: curl -X POST "http://127.0.0.1:8000/generate/definition" \
#           -H "Content-Type: application/json" \
#           -d "{\"user_input\": \"사용자는 이메일과 비밀번호로 로그인할 수 있어야 한다.\"}"
# 2. MiniLM 자연어 모델 임베딩으로 사용자의 입력을 벡터화 (*다른 모델들과의 성능 비교 필수*)
# 3. 사용자 입력 기반 수정이 끝날 때까지 MongoDB에 임시 저장
# 4. 사용자가 "완료"를 선언하면 최종 기능 정의서 확정 (<- 직접 판단하도록 수정할 예정)
# 5. GPT-4o API를 사용해서 기능 정의서로부터 기능 명세서 생성
# 6. 생성 결과를 사용자에게 공유하고, MongoDB에 저장
